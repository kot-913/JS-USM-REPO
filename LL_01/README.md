### 1.3 Первая HTML-страница с JavaScript

Создан файл: `index.html`

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <title>Привет, мир!</title>
  </head>
  <body>
    <script>
      alert("Привет, мир!");
      console.log("Hello, console!");
    </script>
  </body>
</html>
```

**Результат при открытии в браузере:**

- Появилось окно alert с сообщением "Привет, мир!"
- В консоли отобразилось: `Hello, console!`

### 1.4 Подключение внешнего JavaScript-файла

Создан файл: `script.js`

```js
alert("Этот код выполнен из внешнего файла!");
console.log("Сообщение в консоли");
```

Подключение в `index.html`:

```html
<script src="script.js"></script>
```

**Результат:** Сначала сработал alert из внешнего файла (так как `<script>` находится в `<head>`), затем выполнился inline-скрипт в `<body>`.

## Контрольные вопросы

### 1. Чем отличается `var` от `let` и `const`?

**`var`** имеет функциональную область видимости, всплывает и инициализируется как `undefined`, разрешает повторное объявление и переприсваивание.

**`let`** имеет блочную область видимости `{}`, всплывает, но НЕ инициализируется (temporal dead zone), запрещает повторное объявление, но разрешает переприсваивание.

**`const`** имеет блочную область видимости `{}`, всплывает, но НЕ инициализируется, запрещает и повторное объявление, и переприсваивание.

Пример:

```js
var x = 1;
var x = 2; // OK — var разрешает повторное объявление

let y = 1;
// let y = 2;    // Ошибка — let не разрешает повторное объявление

const z = 1;
// z = 2;        // Ошибка — const не разрешает переприсваивание
```

### 2. Что такое неявное преобразование типов в JavaScript?

Неявное преобразование типов — это автоматическое преобразование значений из одного типа в другой при выполнении операций, без явного указания программиста.

```js
"5" + 3; // "53"  — число 3 преобразуется в строку (конкатенация)
"5" - 3; // 2     — строка "5" преобразуется в число (вычитание)
true + 1; // 2     — true преобразуется в 1
"5" * "2"; // 10    — обе строки преобразуются в числа
null + 1; // 1     — null преобразуется в 0
undefined + 1; // NaN   — undefined не может быть преобразован в число
```

Оператор `+` особенный: если один из операндов — строка, выполняется конкатенация. Другие арифметические операторы (`-`, `*`, `/`) всегда пытаются выполнить числовое преобразование.

### 3. Как работает оператор `==` в сравнении с `===`?

- `==` (нестрогое сравнение) — сравнивает значения **с приведением типов**
- `===` (строгое сравнение) — сравнивает значения **и типы без приведения**

```js
5 == "5"; // true  — строка "5" приводится к числу 5
5 === "5"; // false — разные типы (number и string)

0 == false; // true  — false приводится к 0
0 === false; // false — разные типы (number и boolean)

null == undefined; // true  — специальное правило в спецификации
null === undefined; // false — разные типы

"" == 0; // true  — пустая строка приводится к 0
"" === 0; // false — разные типы
```

Всегда используйте `===` (строгое сравнение), чтобы избежать неожиданного поведения из-за приведения типов.
